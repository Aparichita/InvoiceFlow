Identify and fix issues causing blank pages on deployment

Deployment Configuration Check

Needs Attention
  • ✓
    Project Structure

    Your project folder organization looks appropriate

  • !
    Entry Point Configuration

    Check if your package.json has the correct main entry point

    Solution:

    Ensure your package.json has:

    { "main": "index.js", "scripts": { "start": "node index.js", "dev": "nodemon index.js" } }
  • !
    Vercel Configuration

    Check your vercel.json for proper routing settings

    Solution:

    Your vercel.json should include:

    { "version": 2, "builds": [ { "src": "index.js", "use": "@vercel/node" }, { "src": "public/**/*", "use": "@vercel/static" } ], "routes": [ { "src": "/(.*)", "dest": "/index.js" } ] }
  • !
    Environment Variables

    Ensure all environment variables are set in Vercel dashboard

    Solution:

    In your Vercel project dashboard, go to Settings → Environment Variables and add:

    • DATABASE_URL
    • STRIPE_SECRET_KEY
    • JWT_SECRET
    • WHATSAPP_API_KEY
    • Any other API keys your app needs

Server Configuration

Needs Attention
Port Binding
Static Files
Middleware

Vercel requires your app to listen on the port provided by the environment variable PORT.

// In your index.js or app.js const express = require('express'); const app = express(); const PORT = process.env.PORT || 3000; // ... your routes and middleware ... // Start server app.listen(PORT, () => { console.log(`Server is running on port ${PORT}`); });

Ensure your static files are properly served from the public directory:

// Serve static files app.use(express.static('public')); // For client-side routing (like React Router) app.get('*', (req, res) => { res.sendFile(path.join(__dirname, 'public', 'index.html')); });

Check your middleware order and CORS configuration:

// Enable CORS const cors = require('cors'); app.use(cors()); // Body parsing middleware app.use(express.json()); app.use(express.urlencoded({ extended: true })); // Your routes app.use('/api/auth', require('./routes/auth.routes')); app.use('/api/invoices', require('./routes/invoice.routes')); // ... other routes ...

Troubleshooting Steps

Action Required
  1. Check Vercel deployment logs for errors
  2. Test your app locally with production settings: NODE_ENV=production npm start
  3. Add error handling middleware to your Express app
  4. Verify all file paths are correct (case-sensitive on Vercel)
  5. Check that all required modules are in package.json dependencies (not devDependencies)

Error Handling Middleware Example:

// Add this at the end of your middleware chain app.use((error, req, res, next) => { console.error(error); res.status(500).json({ message: 'Internal Server Error', error: process.env.NODE_ENV === 'development' ? error.message : {} }); });
